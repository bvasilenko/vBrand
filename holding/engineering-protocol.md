# Engineering Protocol

## Post-deploy chief-executive eye-test gate (C-152)

After every `npm publish` + `gh-pages` deploy, the `dist/eye-test.png` screenshot grid is the
mandatory out-of-pipeline verification surface before declaring cycle close.

**Artefact:** `dist/eye-test.png` — 60-cell grid (5 fixtures x 4 app-types x 3 modes).
Generated by `bun run eye-test:grid` against the live hosted surface.

**Companion artefact:** `dist/eye-test-manifest.json` — per-cell metadata (primary hex,
stylesheet count, island count, automated diff flags).

**Gate:** Supervisor attaches `dist/eye-test.png` to the cycle close log and shares the hosted
preview URL before requesting chief-executive ratification. Local `bun run preview` verification
is insufficient (C-139). Contribot-internal web-qa verdict is insufficient (C-152).

**Retrospective lock-in:** C-152 was introduced because alpha.4 shipped with chief-executive-visible
bugs despite a CLEAN Playwright probe verdict -- the probe gate verifies behavioral contracts;
the screenshot grid catches visual regressions that probes cannot express.
